# grass

Vite + React 19 + React Three Fiber demo: 72k instanced grass blades on a 3x3 tile grid.

## Structure

- `src/App.jsx` — Canvas, scene composition, ground plane, soil block. Also the `Flipper` coin-flip rig: the whole grass diorama is wrapped in a group pivoting π about x at slab mid-height (y=-1), with `<City>` mounted pre-rotated on the slab's underside so the arrow button (`.flip-btn`, bottom-right HTML overlay) somersaults the block and lands downtown Detroit upright in the same footprint. The transition is a progress timeline (`p` 0→1 over ~1.7s), not a damp: `easeFlip` (gentle easeInOutBack) gives a wind-up tip and a settling over-rotation, a smoothstepped sin hop lifts the block mid-flip, the sky palette crossfades from the current leva "background" colors (captured at rest) to a steely `CITY_SKY`, and a dust ring puffs at either landing. Writes `flipState.p` each frame; whichever side is face-down is culled at rest so only one scene renders. Opening the app with `#city` in the URL starts on the Detroit side (no flip animation).
- `src/city/flipState.js` — `{ p }` module singleton: App's Flipper writes flip progress, the city's light rig reads it.
- `src/city/City.jsx` — downtown Detroit diorama, authored ground-at-y=0 (App's wrapper handles the flip math). All static architecture (slab, sidewalk ground, streets/crosswalks, ~22 generic buildings with window grids, RenCen, Penobscot, Guardian, One Detroit Center, Book Tower, Fox sign, Spirit of Detroit, Dodge Fountain, Comerica Park, Ford Field, Ambassador Bridge, trees, streetlights, plus a wordless riverfront civic plaza with an abstract automotive wheel sculpture, landscaped seating, colorful crowds, parasols, and a food truck) is a Blender-authored GLB (`public/detroit.glb`) with Cycles lighting baked into two high-resolution atlases; `BakedCity` loads it via drei `useGLTF` inside its own `<Suspense>` and swaps every mesh to unlit `MeshBasicMaterial` (lighting lives in the texture — regenerate with `tools/blender/`). Still procedural here because they animate or must match three.js curves exactly: the river shader (gouache bands/foam/glints on shared `uTime`), People Mover guideway + pylons + two-car train (`getPointAt`/`getTangentAt` on the CatmullRom curve), looping traffic + DDOT bus, freighter + sailboat, Penobscot beacon + fountain-jet pulses (`CityPulse`), the People Mover stations, and the riverwalk railing (too thin to bake — atlas islands smear). All dynamic pieces live in the GLB's authored coordinate frame (river/water at +z 5.19..7.45, the near-left foreground edge; City-local == world when the city side is up — the premount π and the Flipper's π cancel); traffic runs only where the radial plan has straight lanes — Jefferson (x-axis lanes, z 3.7..4.2) and Woodward north of Campus Martius (z-axis lanes, x ±0.15, z −0.2..−7.4). The ambient+directional rig fades in with `flipState.p` and only lights those Lambert dynamic pieces.
- `tools/blender/` — the bake pipeline that generates `public/detroit.glb` (see its README): drives Blender through the blender-mcp addon socket (`bmcp.py`), builds the whole scene in three.js coordinates (`P(x,y,z)→(x,-z,y)`, glTF export converts back), joins into 2 atlas objects, smart-UVs, Cycles-GPU-bakes COMBINED (sun + world + emissive windows), exports the GLB.
- `src/StyleEffect.jsx` — the only post-processing: one `postprocessing` Effect with a `uMode` switch behind a "style" leva dropdown (none / ink / anime / outline / b&w) plus line-strength and color-band sliders. All modes share a 3×3 sobel on alpha-weighted luma; `none` unmounts the EffectComposer entirely. Replaced the old Bloom + Kuwahara stack.
- `src/Camera.jsx` — orthographic isometric corner view at [12.5,11,12.5] targeting (0,-1,-0.4): the square tile reads as a floating diamond diorama (target below ground keeps the soil block's underside in frame). Aspect-aware `view` half-height with clamps; OrbitControls constrained to a narrow polar/azimuth window around the corner angle.
- `src/coast.js` — single source of truth for the in-tile coastline: `COAST_EDGE` (grass is fully dune sand by z=-COAST_EDGE), `LAND_EDGE_Z` (ground plane discards past this; the ocean plane below shows through), `WATERLINE_Z`, plus the shared sand/dune-grass colors + uniforms that keep ground, grass, ocean, and soil block sand identical. The grass vertex shader, ground fragment, and SoilBlock all interpolate these constants.
- `src/grass/constants.js` — GRID/TILE/blade counts/param-texture layout constants.
- `src/grass/shaders.js` — GLSL: wind bend + hash-cell clump pull (uClump, cell size from uClumpScale, mound-shaped height taper toward each clump center) + param fetch in vertex, fbm color fields + gust sheen (vGust varying) + AO in fragment. Interpolates constants into the source.
- `src/grass/geometry.js` — `buildGeometry()`: one normalized segmented strip as `InstancedBufferGeometry` with per-blade attributes (offset, scale, rotation, wind phase, tile id). The vertex shader shapes it with live blade-profile uniforms. Blades are laid out blade-major (tiles innermost) so the "blades" slider can truncate `instanceCount` and thin all tiles evenly.
- `src/grass/tileParams.js` — the per-tile `DataTexture` (4 RGBA float texels per tile: colors A/B/C + scales/mixes) and `randomizeTiles()` which rewrites it with hue/sat/lightness-jittered colors. Jitter is seeded (mulberry32) so slider changes re-tint without reshuffling the layout; only `reseed()` (the "randomize" button) rolls a new arrangement.
- `src/grass/material.js` — the `uniforms` object + ShaderMaterial, built manually as module singletons — R3F's `uniforms` prop clones each uniform into the material, so mutating a shared uniforms object via JSX `<shaderMaterial>` never reaches the GPU. Exported so App.jsx's "ground / blade base" control can drive `uBaseColor` too (ground plane and blade roots share one color). Also exports `groundMaterial`: samples the density mask for pressed-turf/soil tint, embosses two-scale noise on the soil for path bumpiness, and reuses the shared `uTime`/`uGustScale`/`uSheen` uniform objects to run the same traveling gust-sheen formula as blades/rocks so the wash lightens the path in sync.
- `src/grass/densityMask.js` — 256² CPU-baked `DataTexture` carving a meandering dirt path: R = surviving blade density, G = pressed-grass amount (also darkens the ground), B = exposed-soil patches. Sampled by both the grass vertex shader (kills/shortens blades) and `groundMaterial` (pressed-turf + soil tint). `setPathParam(key, value)` re-bakes the whole texture (width/soil/clearing/core), driven by the "path" leva folder.
- `src/grass/Grass.jsx` — the mesh + leva controls (`useControls`); color/scale changes and the "randomize" button call `randomizeTiles`. A "wind" folder (strength/speed/gust scale) writes transiently to the `uWindStrength`/`uWindSpeed`/`uGustScale` uniforms via `onChange` — no re-render. A "path" folder mixes both patterns: width/soil/clearing/core call `setPathParam` (CPU mask re-bake), press/soilColor/darken/bump/bumpScale write to uniforms. Path defaults live in `PATH_DEFAULTS` in `defaults.js`.
- `src/rocks/rockMask.js` — 256² world-space occupancy `DataTexture` (`uRockMask` in the grass material); keeps two footprint lists — `stampRockMask(footprints)` (re-stamped on every rock scatter) and `stampSceneryMask(footprints)` (static, from Scenery.jsx) — merged into one bake so neither wipes the other. Writes soft-rimmed circles (solid kill under the rock, fading out to `RING`× its radius) and the grass vertex shader scales blades down by the mask — grass never renders through rocks and ramps up gradually around each base so rocks look nestled in.
- `src/Ocean.jsx` — Ghibli-style beach + ocean INSIDE the tile along the -z edge (where the dirt path exits): one flat plane exactly as wide as the field, spanning from just inland of `LAND_EDGE_Z` to the tile edge, tucked below the ground plane (the ground discards past `LAND_EDGE_Z`, so the ocean shows through there). Everything painted in the fragment shader keyed off signed distance past a breathing waterline (run-up/retreat + wobble) — mottled dry sand (world-space formula identical to the ground's dune sand, hiding the discard seam) → wet sand with a foam-arc "ghost" → scalloped foam + broken second line → gouache-banded turquoise-to-teal compressed into the short in-tile band. Vertex bob on open water only. Reuses the shared `uTime` uniform object from `grass/material.js`, so Grass's frame loop animates it — no `useFrame` here. Module singleton, no controls.
- `src/SoilBlock.jsx` — diorama base: a 15×2×15 box under the tile painted entirely in-shader — mottled soil gradient with wavy sediment strata, sparse embedded pebbles plus larger irregular stones (lit top / shaded belly), hanging root tendrils and fine root hairs descending from the lip (faded where the rim is sand), root-shadow band, a ragged grass lip that turns into a deeper sand wash along the coast edges, and the path's sand column carried down the +z face. Unlit; sits below the ground/ocean planes. Module singleton, no controls.
- `src/Scenery.jsx` — set dressing: a gabled cottage on the grass above the dune, stepping stones from its door to the path, and the beach-lounging vignette on the dry-sand band before the waterline — placements in `HOUSE`/`CAMP`/`STONES` are keyed to the in-tile coastline in `coast.js`. The two figures have full limbs (lying figure: one straight leg + one bent knee-up with feet; sitting figure: feet flat on the sand and hands where the propped arms land), distinct skin tones (lying: `skin`, sitting: deeper `skinTan`/`skinTanShadow`), shoulder-joint spheres where the arms meet each torso, swimsuit straps on the lying figure's chest, and a simple face on the sitting figure. The sitting figure's arms/legs are `Limb`/`Segment` joint-point chains — capsules stretched between shared joint coordinates, so segments always connect and pose edits mean moving joint points, not re-deriving capsule transforms; joint spheres are skin-colored (not shadow-toned) on both figures so elbows/knees don't read as mannequin balls. Face details: sitting figure — dark oval eyes, blush dots, nose, mouth — plus a low hair bun poking out behind the head under the hat brim, and a gentle idle `useFrame` in `Loungers` — breathing chest swell/lift on both torsos, the sun hat rocking with the lying figure's breath, and the sitting figure's head (grouped with its hair so they pivot together) slowly scanning the horizon with a breath nod. Cottage build: (gable-end pentagon and chevron roof slab both built as `ExtrudeGeometry` along the ridge — eaves/gable overhang/roof thickness come from the profile, no rotated-box math; ridge-cap cylinder along the peak, timber-framed gable (wall-height beam + raking beams) on the door end, arched extruded door + frame with knob, shuttered windows with mullion cross + sill and a flower box, round gable window, chimney with `useFrame`-animated smoke puffs), stepping stones toward the dirt path, and a beach-lounging vignette near the path exit (striped towel + lying figure with limbs and a straw sun hat over the face, sitting figure with knees-up legs / propped arms / hair cap, leaning parasol with an 8-wedge two-color striped canopy and finial). Plain Lambert materials lit by the rig in Rocks.jsx; no controls. Calls `stampSceneryMask` at module load so grass doesn't grow through footprints.
- `src/Birds.jsx` — 5 low-poly gulls circling above the field (capsule body, sphere head, orange beak, forked shape-geometry tail, two-segment wings with a swept black-tipped outer panel; single white plumage, Lambert-lit like the scenery). Each follows a per-bird Lissajous path whose amplitudes are capped below `FIELD_HALF`, so flight is bounded to the tile by construction — no steering/boids. `useFrame` samples the path at t/t+0.25/t+0.75 for position, heading (`lookAt`), and banked roll into turns; wings beat with a skewed (fast-upstroke) sine, the outer panel trailing the inner in phase, eased into intermittent glides by a smoothstep envelope biased by vertical velocity (beat uphill, glide downhill), with a downstroke body bob. The head (neck-pivot group) idly scans, looks into turns, and tilts down in glides; the tail rudder-yaws against turns, fans up in glide, and flicks with the beat. Each bird drags a blob contact shadow (radial-gradient `CanvasTexture` disc just above the ground, per-bird cloned basic material) that tracks x/z and grows/fades with altitude — the ground/ocean shaders are unlit so real shadow maps can't reach them. Mirrored right wing: only the `scale.x=-1` node's own rotation.z negates — nested rotations keep their sign. Module-singleton geometry/params, no controls.
- `src/rocks/Rocks.jsx` — self-contained generative rock system: one `InstancedMesh` (module singleton, max 80) of a seeded sine-displaced, flat-shaded icosahedron with a 3-band `MeshToonMaterial` for the cel look, plus the ambient/directional lights that feed it (grass/ground materials ignore lights). A "rocks" leva folder drives count/size/variation/jaggedness/squash/spread plus a two-color gradient (color A/B, with a "gradient" slider scaling how much of the A→B span is sampled) each rock samples at a random point for its tint; seed-gated like the grass — sliders re-scatter the same layout, the "randomize" button rolls a new seed. Scatter rerolls positions that land past the in-tile coastline so rocks stay off the beach/water. `onBeforeCompile` injects the grass gust-sheen (same noise formula, shared uniform objects from `grass/material.js`) so gusts wash over rocks and grass together, plus a moss layer: world-space noise patches masked to up-facing surfaces, mixed into `diffuseColor` before the toon ramp so moss shares the cel bands. A "moss" sub-folder (amount/patch scale/color) writes transiently to module-level `mossUniforms` via `onChange` — no re-render, no re-scatter.

## Conventions

- Geometry, param texture, and uniforms are module-level singletons — the React Compiler lint forbids render-scope mutation, and there's only one `<Grass>`. Move them into the component if it ever mounts twice.
- `npm run lint` uses the React Compiler eslint rules — strict about purity/mutation during render.
